Custom Property

Custom properties are custom fields for resources like boards, tasks, etc. Usually defined in the resource's parent.
- Custom properties can be defined in organizations, for boards
- Can be defined in boards for tasks
- Should be defineable for other sub fields including custom properties and subtasks, etc.

Work focus:
- Defining custom properties in boards for tasks
- Defining custom properties in organizations for boards
- Defining organization-wide custom properties reused accross the board
- Cloning boards to copy over it's custom properties
- Getting custom properties from other boards
- Moving some existing app-provided properties like status and labels to custom properties
- Implemeting one size fits all custom properties endpoints

Endpoints:
- createProperty
- updateProperty
- deleteProperty
- getProperties
- getValues
- updateValue
- createCustomSelectionOption
- updateCustomSelectionOption

createProperty
- validates property
- check that the user has permission to create the property in the org, 
  and that the direct parent exists
- check that the property name does not exist
- save property

updateProperty
- validate property
- check permission
- assert check property
- save property
- delete previous values if type is changed

deleteProperty
- check permission
- delete property
- delete values

getProperties
- validate
- check permission
- return properties

getValues
- validate
- check permission
- return values

updateValue
- validate
- check permission
- update value

createCustomSelectionOption
- validate
- check permission
- check property exists
- check property is custom
- check unique if options should be unique
- save option

updateCustomSelectionOption
- validate
- check permission
- check unique if options should be unique
- save option
